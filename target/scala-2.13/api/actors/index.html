    The game actor is an Akka Actor that receives events from the user front-end UI (e.g. when the user clicks on the board) via a websocket connection. When an event arrives, the processMessage() method is called, which can be used to react to the event. The Game actor also includes an ActorRef object which can be used to issue commands to the UI to change what the user sees. The GameActor is created when the user browser creates a websocket connection to back-end services (on load of the game web page).
