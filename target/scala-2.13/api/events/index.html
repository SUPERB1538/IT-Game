  4.  class Heartbeat extends EventProcessor

    In the user’s browser, the game is running in an infinite loop, where there is around a 1 second delay between each loop.

    In the user’s browser, the game is running in an infinite loop, where there is around a 1 second delay between each loop. Its during each loop that the UI acts on the commands that have been sent to it. A heartbeat event is fired at the end of each loop iteration. As with all events this is received by the Game Actor, which you can use to trigger game logic.

    { String messageType = “heartbeat” }
